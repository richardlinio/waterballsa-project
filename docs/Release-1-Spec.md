# Release 1 規格清單 v03

- Last update: 2025-11-18

## 必做規格

### 1. 權限模組

#### 1.1 使用者註冊

- 使用者可以使用帳號、密碼註冊
- 帳號應為英數字且有長度上限、密碼應為英數字與特殊符號且有長度上限，大小寫均不限
- 起始經驗值為 0
- 註冊成功後導向登入頁面
- 錯誤處理：帳號已存在時，不給註冊重複帳號

#### 1.2 使用者登入

- 使用者用帳號和密碼登入
- 登入成功後顯示課程列表和使用者帳號、經驗值
- 錯誤處理：帳號或密碼錯誤，不給登入

#### 1.3 登入狀態保持

- 使用者重新整理頁面後仍保持登入
- 關閉瀏覽器後再開啟仍保持登入
- 過期後（預計一天）需要重新登入

#### 1.4 使用者登出

- 使用者可以主動登出
- 登出後無法訪問需要登入的頁面

### 2. 看課模組

#### 2.1 課程列表與瀏覽

- 顯示所有課程（封面圖、標題、簡介、價格）

#### 2.2 課程詳情頁

- 顯示課程完整資訊（標題、描述）
- 顯示章節與單元結構
- 單元可點擊，進到該影片單元

#### 2.3 影片觀看與進度追蹤

- 使用者可以播放課程影片
- 觀看進度達 100% 視為完成
- 影片完成度必須自動追蹤，不能讓使用者手動 mark 成完成狀態
- 使用者可以重複觀看已完成單元
- 系統定時記錄觀看進度，暫定每 10 秒記錄一次
- 下次觀看時從上次位置繼續播放

#### 2.4 交付功能

- 影片單元觀看進度達到 100% 之後，影片單元的狀態就會變成「已完成」
- 已完成的單元旁邊會有個灰色小圈圈 UI
- 學員點擊圈圈就可以交付單元，獲得該單元的經驗值，UI 變成綠色
- 經驗值一單元 +100（MVP 先寫死）
- 每個單元只能交付一次

#### 2.5 經驗值系統

- 個人頁面可檢查經驗值
- 帳號初始經驗值為 0
- 交付課程單元獲得的經驗值由課程單元自行定義，先假設每個單元有自己的經驗值，MVP 先用固定值（例如 100）
- 作為未來排行榜順序依據

## 延後執行規格

### 1. 權限模組

#### 1.1 使用者註冊

- 系統驗證帳號格式和密碼強度
- 註冊成功後自動登入並導向課程列表

#### 1.2 使用者登入

- 數次登入錯誤會鎖一小段時間，避免暴力破解
- 記住帳號，下次登入自動帶入帳號
- 忘記帳號，可取回帳號
- 忘記密碼，可取回密碼

#### 1.3 登入狀態保持

- Access token 與 refresh token 機制，降低 token 竊取攻擊面

#### 1.4 使用者登出

- 登出後 token 加入黑名單，避免被竊取後由第三人登入

### 2. 看課模組

#### 2.1 課程列表與瀏覽

- 篩選功能：全部/已購買/未購買

#### 2.2 課程詳情頁

- 顯示課程完整資訊（標題、描述）
- 顯示章節與單元結構
- 已購買課程：單元可點擊，顯示完成狀態
- 未購買課程：單元顯示鎖定圖示
