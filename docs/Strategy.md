# 水球軟體學院面試挑戰 - 10 天作戰策略

## 作戰分析

### 關鍵洞察

1. **評分標準的真相**：他們明確說「不期待全部完成」，重點是：

   - **Discovery**：主動釐清需求的能力
   - **Formulation**：規格化能力
   - **Automation**：AI 輔助開發能力
   - **多次 Release**：敏捷迭代能力

2. **致命要求**：

   - **必須主動提問**（不提問直接落選）
   - **必須有 E2E 測試**
   - **必須多次 Release**

3. **技術債風險**：
   - 需要熟悉 Spring Boot + Next.js
   - 時間只有 10 天
   - 還要學 E2E 測試

---

## 核心策略：聰明的取捨 + AI 最大化

### 策略原則

1. **不要做完美主義者**：功能「能動」比「完美」重要
2. **善用 AI**：讓 Claude/GPT 寫 80% 的 boilerplate code
3. **刷存在感**：頻繁在 Discord 提問和 Release
4. **展現思維**：讓面試官看到你的規劃能力

---

- **Release 1**：Day 5 完成權限 + 看課完整功能

## 關鍵成功因素

### 1. Discord 互動策略

**頻率**：至少每天 1-2 次提問或 Release

**質量問題範例**：

- 「影片觀看進度是即時儲存還是離開頁面才儲存？」
- 「交付後經驗值是立刻發放還是需要審核？」
- 「排行榜是即時更新還是每日更新？」
- 「道館挑戰的兩種模式（個人/團體）在系統中有什麼差異？」
- 「獎勵任務的機會卡消耗邏輯是什麼？」

### 2. AI 使用技巧

**用 AI 生成**：

- Spring Boot Entity/Repository/Service
- Next.js 頁面 boilerplate
- E2E 測試腳本
- Docker Compose 配置
- 資料庫 Migration 腳本

**自己寫的部分**：

- 核心業務邏輯
- 資料庫 Schema 設計
- API 設計
- 重要的邊界條件處理

### 3. 技術取捨建議（依 PM 確認優先級）

#### ✅ 必須做（Release 1 & 2）

**Release 1 核心**

- 使用者註冊/登入（JWT）
- 課程列表 + 課程詳情
- 影片播放 + 觀看進度追蹤
- 交付功能 + 累積經驗值
- E2E 測試（至少涵蓋核心流程）

**Release 2 核心**

- 購課流程（**簡化版金流**，用按鈕模擬，不串真實支付）
- 購買後解鎖課程
- 經驗值計算 + 等級系統
- 排行榜（依經驗值排序）

#### ⚠️ 簡化做

- 影片播放（用 YouTube embed）
- 購課金流（按鈕模擬即可）
- 訂單記錄（簡單列表即可）
- 個人檔案（基本資訊 + 經驗值）

#### ⏸️ 延後實作（Day 8+ Optional）

- 道館挑戰
- 作業系統
- 文章/問卷
- 獎勵任務

#### ❌ 不要做

- Clean Architecture（他們明確說不要）
- 完美的 UI（能用就好）
- 真實金流串接（PM 明確說簡化）
- 單元測試（隨意，專注 E2E）
- 雲端部署（本地 Docker 就好）
- 複雜的權限系統（基本 JWT 即可）
- 即時通知功能

### 風險 2：E2E 測試沒時間

**對策**：

- ✅ Day 2 就建立測試框架（已完成）
- 每開發一個功能就寫對應測試
- 用 AI 生成測試腳本

**實際狀況**：

- Playwright 已初步建置，進度符合預期

### 風險 3：功能做不完

**對策**：

- **這是預期的！** 他們不期待你做完
- 重點是展現「規劃能力」和「迭代思維」
- 寧可 4 個功能完整，不要 10 個功能半殘
- 確保每個 Release 都是可 Demo 的狀態

**實際應對**：

- 新增 Release 0（非正式里程碑）提早與 PM 確認
- 將 Release 1 目標從 Day 5 調整為 Day 6，更務實
- 影片托管為技術風險，需於 Day 2 晚上確認方案

### 風險 4：Docker 環境問題

**對策**：

- ✅ 盡早測試 Docker Compose（已完成）
- 準備本地開發和 Docker 兩套環境
- 記錄所有環境設定步驟

**實際狀況**：

- Docker Compose 已建立並可正常運作

### 風險 5：影片托管技術（新增）

**識別原因**：

- PM 明確要求「每 10 秒自動記錄觀看進度」

- YouTube embed 是否支援此功能需驗證

**對策**：

- Day 2 晚上進行技術驗證

- 若不可行需評估其他方案（如自建影片播放器）
- 必要時向 PM 確認是否可調整技術實作方式

## 成功心態

1. **這不是技術考試，是軟體工程思維考試**
2. **溝通比完美的 code 更重要**
3. **多次小勝利（Release）勝過一次大失敗**
4. **主動提問展現你的思考深度**
5. **善用 AI 是展現能力，不是作弊**

### 你的最大優勢

不是技術純熟度，而是：

1. **展現規劃思維**：讓他們看到你會切 Release
2. **主動溝通**：刷 Discord 存在感
3. **善用 AI**：10 天做出 1 個月的產出
4. **快速學習**：展現你能快速上手新技術
